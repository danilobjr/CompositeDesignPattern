Composite

O que é e por que usar?

Composite


O que é?

  • Padrão de Projeto usado para representar um objeto que contenha uma lista de objetos do mesmo tipo.

Por que usar?

  • Permite uma inclusão dinâmica, em tempo de execução, de objetos-filho em níveis previamente não definidos.

Código

Composite no C# 

    interface IComposite
    {
        IList<IComposite> Filhos;
        String Nome { get; set; }
       
        void Adicionar(IObjeto o);
    }

    class Arquivo : IComposite
    {
        public IList<IComposite> Filhos { get; set; }
        public String Nome { get; set; }

        public Arquivo(String nome) { this.Nome = nome; }
        
        public void Adicionar(IComposite elemento) { Console.Write("não permitido"); }
        public override string ToString() { return this.Nome; }
    }

Código

Composite no C# 

    class Pasta : IComposite
    {
        IList<IComposite> Filhos { get; set; }
        public String Nome { get; set; }

        public Pasta(String nome)
        {
            this.Nome = nome;
            this.Filhos = new List<IComposite>();
        }

        public void Adicionar(IComposite elemento) { this.Filhos.Add(elemento); }
        public override string ToString() { return this.Nome; }
    }
